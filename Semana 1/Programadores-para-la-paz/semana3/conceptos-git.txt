Repositorio: Básicamente es la carpeta digital donde guardamos todo el proyecto de forma organizada: los archivos, el código y, lo más importante, todo el historial de cambios. Hagamos de cuenta que funciona como la base de datos de nuestro código, ahí es donde Git controla y sigue cada pequeña modificación o línea que editamos a lo largo del tiempo para que nunca se nos pierda nada. 
Commit: Es como tomarle un pantallazo o una foto al estado actual del código para guardar los cambios en el historial del PC. Cada commit funciona como un punto de restauración, si nos equivocamos más adelante, podemos volver a esa foto exacta. 
Push: Este es el comando que manda todos los commits que tenemos guardados en nuestra computadora hacia el repositorio remoto en la nube, o sea a GitHub. Sirve para que el código tenga un respaldo seguro en internet por si a nuestro PC le pasa algo, y que las demas personas del equipo puedan ver, descargar y trabajar sobre las últimas actualizaciones que subimos.
