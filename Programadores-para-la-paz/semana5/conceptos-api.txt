Análisis de la ruta POST /reporte:

1. ¿Qué hace esta ruta?

Esta ruta escucha peticiones HTTP bajo el método POST en el endpoint '/reporte'. A diferencia de las rutas GET (que solo sirven para pedir o ver información), POST se usa para enviar datos hacia el servidor de forma segura. El servidor recibe un JSON con un 'mensaje' dentro del cuerpo de la solicitud (req.body), lo procesa y responde devolviendo un objeto que confirma que el estado es "Reporte recibido" junto con el texto original que envió el usuario.

2. ¿Para qué podría servir en una aplicación comunitaria?
En una app para el barrio o la comunidad, esta ruta es clave para la participación de la gente. Podría servir para:

- Crear un formulario digital donde los vecinos reporten incidentes reales (por ejemplo: una luminaria pública dañada, un bache en la calle o basura acumulada).

- Enviar alertas comunitarias inmediatas sobre situaciones de riesgo o emergencias en una zona específica.

- Registrar propuestas, quejas, reclamos o incluso votos en encuestas vecinales organizadas.

Al usar POST, los reportes viajan ordenados y limpios directo a la base de datos sin generar el caos y desorden de los grupos de WhatsApp o Facebook.
