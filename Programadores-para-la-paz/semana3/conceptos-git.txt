Repositorio: Hagamos de cuenta que es la carpeta digital donde ingresamos todo el proyecto para que no quede organizado: los archivos, las líneas de código y todo el historial. Funciona basicamente como la base de datos, ahí es donde Git rastrea cada cambio o línea que editemos en el tiempo para que nunca, se nos vaya a perder nada.
Commit: Es como jugar una campaña bien difícil y acabas de pasar una misión dificil o de matar a un jefe, el commit es como el punto de guardado que hacemos antes de seguir avanzando. Cada commit es un registro seguro en el historial del PC. Si más adelante en el siguiente nivel nos equivocamos, dañamos un código, no perdemos todo, sino que nos devolvemos a ese último punto guardado y ya. Siempre dejamos una etiqueta o mensaje breve para saber qué misión o arreglo dejaste listo en ese punto.
Push: Este es el comando que manda todos los commits que tenemos guardados en el computador directo al repositorio remoto en la nube, o sea a GitHub. Sirve para tener un respaldo seguro en internet por si el pc se nos daña, para que los demás del equipo puedan ver, descargar y trabajar sobre las últimas actualizaciones que subimos sin enredarnos.
